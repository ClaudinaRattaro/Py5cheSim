Py5cheSim.simulation

module documentation

This is the simulation script. Simulation, cell and traffic profile parameters can be set here.

Variable bw List containing each CC's bandwidth for the simulation.
Variable fr String with frequency range (FR) to use. 'FR1' for FR1, or 'FR2' for FR2.
Variable band String with used band for simulation. In TDD mode it is important to set correctly a band from the next list: n257, n258, n260, n261.
Variable tdd Boolean indicating if the cell operates in TDD mode.
Variable buf Integer with the maximum Bytes the UE Bearer buffer can tolerate before dropping packets.
Variable schedulerInter String indicating the Inter Slice Scheduler to use. For only one Slice simulations use ''. If the simulation includes more than one slice, set '' for Round Robin, 'RRp' for Round Robin Plus, or 'PFXY' for Proportional Fair with X=numExp and Y=denExp.
Variable t_sim Simulation duration in milliseconds.
Variable debMode Boolean indicating if debugging mode is active. In that case, an html log file will be generated with schedulers operation. Note that in simulations with a high number of UEs this file can turn quite heavy.
Variable measInterv Time interval (in milliseconds) between meassures for statistics reports.
Variable interSliceSchGr Inter slice scheduler time granularity in milliseconds.
Variable env Environment instance needed by simpy for runing PEM methods
Variable cell1 Cell instance for running the simulation
Variable UEgroup2 Undocumented
Variable UEgroup1 Group of users with defined traffic profile, capabilities and service requirements for which the sumulation will run.
Variable UEgroups UE group list for the configured simulation

UEgroup1 =

Group of users with defined traffic profile, capabilities and service requirements for which the sumulation will run.

More than one can be instantiated in one simulation. For each one of them, the UEgroup instance must be added in the UEgroups list.

UEgroupN = UEgroup(nuDL,nuUL,pszDL,pszUL,parrDL,parrUL,label,dly,avlty,schedulerType,mimo_mode,layers,cell,hdr,t_sim,measInterv,env,sinr):

label: must contain substring according to the type of service: eMBB, mMTC, URLLC

schedulerType: RR: Rounf Robin, PF: Proportional Fair (10, 11)

mimo_mode: SU, MU

layers: in SU-MIMO is the number of layers/UE, in MU-MIMO is the number of simultaneous UE to serve with the same resources

sinr: is a string starting starting with S if all ues have the same sinr or D if not. The value next will be the initial sinr of each ue or the maximum.

UEgroups =

UE group list for the configured simulation
